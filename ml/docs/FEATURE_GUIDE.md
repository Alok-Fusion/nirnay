# Feature Engineering Guide

This guide details the features generated by the `ml.feature_engineering.pipeline` module from the raw relational datasets.

## Feature Categories

### 1. Amount & Balance Features
- `amount`: Raw transaction amount.
- `amount_deviation`: Ratio of current `amount` to the customer's `average_transaction` baseline. High deviations (>3x) heavily indicate risk.
- `balance_ratio`: Ratio of `amount` to `current_balance`. Values near 1.0 (emptying the account) are common in investment and romance scams.

### 2. Temporal & Behavioral Features
- `hour_of_day`, `day_of_week`, `day_of_month`: Extracted from transaction timestamp.
- `is_weekend`, `is_night_transaction`: Binary flags. Scams often occur during off-hours when victims cannot contact bank branches.
- `salary_week`: Binary flag indicating if the transaction occurs during typical salary deposit periods (1st-7th or 28th-31st).

### 3. Recipient & Merchant Trust Features
- `is_trusted_recipient`: Binary flag indicating if the recipient is saved/trusted.
- `recipient_familiarity`: Continuous variable representing historical transaction count with the recipient.
- `merchant_trust_score`: 0-100 score indicating merchant reputation.
- `merchant_scam_reports`: Number of scam flags associated with the merchant.

### 4. Deviation Flags
- `is_favorite_category`: Binary flag matching the transaction category to the user's `favorite_categories`.
- `location_deviation`: Binary flag if `device_location` != `usual_location`.
- `device_deviation`: Binary flag if `device_type` != `usual_device`.

### 5. Profile Attributes
- `customer_risk_profile_score`: Ordinal mapping of the customer's baseline risk (Low=1, Medium=2, High=3).
- `digital_payment_ratio`: 0.0-1.0 representing comfort with digital banking.

## Preprocessing Details
- Handled by `ml.preprocessing.pipeline`.
- **Numerical Features**: Scaled using `StandardScaler`. Missing values imputed with median.
- **Categorical Features**: One-Hot Encoded (e.g. `transaction_type`, `payment_channel`, `category`). Missing values imputed with "Unknown".
